## Answer Key

### Section 1: Multiple Choice

1. **B** - It saves time and ensures consistent quality through reuse
   - *Explanation: Libraries eliminate recreating prompts and provide tested, reliable options.*

2. **C** - By use case or task category
   - *Explanation: Organizing by use case makes prompts easy to find when needed.*

3. **B** - To provide a condensed version for fast copy-paste use
   - *Explanation: Quick Start enables rapid use without reading full documentation.*

4. **B** - Clear documentation of variables and expected output
   - *Explanation: Others need to understand inputs and expected results to use prompts correctly.*

5. **B** - When the prompt no longer works or is relevant
   - *Explanation: Archiving preserves history while removing outdated prompts from active use.*

6. **C** - Descriptive names indicating category and task
   - *Explanation: Descriptive names enable quick identification without opening files.*

7. **C** - Regularly (weekly quick checks, monthly reviews, quarterly deep review)
   - *Explanation: Regular maintenance keeps the library current and useful.*

8. **B** - Variables, expected output, examples, and limitations
   - *Explanation: Complete documentation enables anyone to use prompts effectively.*

9. **B** - A showcase of your best, most effective prompts
   - *Explanation: Portfolios highlight expertise and most valuable work.*

10. **B** - To track changes, enable rollback, and document evolution
    - *Explanation: Version control provides history and the ability to revert changes.*

### Section 2: True/False

11. **False** - Libraries can include working prompts at various quality levels; documentation should note scores.

12. **True** - Examples demonstrate correct usage and expected outcomes.

13. **False** - Archiving preserves history for reference; deletion loses potentially useful information.

14. **True** - Consistent templates improve usability and reduce learning curve.

15. **False** - AI models change; prompts need periodic re-testing to verify they still work.

### Section 3: Scenario-Based

16. **B** - Clearly document each variable with descriptions and sample values
    - *Explanation: Variable documentation solves the confusion directly.*

17. **B** - Organize prompts into category-based subfolders
    - *Explanation: Folder structure enables quick navigation to relevant prompts.*

18. **B** - Test and document which prompts are affected, then update them
    - *Explanation: Systematic assessment and documentation enables targeted fixes.*

19. **B** - Add customization options and documentation for different use cases
    - *Explanation: Customization documentation extends prompt usefulness.*

20. **B** - A README with overview, "most popular" section, and getting started guide
    - *Explanation: Onboarding documentation helps new users navigate large libraries.*
