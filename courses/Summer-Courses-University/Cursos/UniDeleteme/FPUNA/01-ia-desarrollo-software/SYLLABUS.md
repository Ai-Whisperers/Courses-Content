# Syllabus: IA para Desarrollo de Software
## Programa Detallado del Curso

---

## Información del Curso

- **Código**: FPUNA-EXT-IA-DEV-01
- **Créditos**: 1 crédito de extensión
- **Horas**: 16 horas presenciales
- **Prerequisitos**: Programación básica

---

## Módulo 01: Introducción a IA en Desarrollo
**Duración: 1.5 horas**

### Objetivos
- Comprender el estado actual de la IA en desarrollo de software
- Conocer las principales herramientas disponibles
- Entender capacidades y limitaciones

### Contenidos
1. Evolución de las herramientas de desarrollo
2. ¿Qué es un LLM y cómo funciona?
3. Panorama de herramientas: Copilot, Claude, ChatGPT, Cursor
4. Casos de uso y casos de anti-uso
5. Ética y responsabilidad en el uso de IA

### Actividades
- Demo: Comparación de herramientas en vivo
- Discusión: ¿Reemplazará la IA a los programadores?
- Quiz de conceptos básicos

---

## Módulo 02: GitHub Copilot
**Duración: 2 horas**

### Objetivos
- Instalar y configurar GitHub Copilot
- Dominar atajos y flujos de trabajo
- Maximizar productividad con Copilot

### Contenidos
1. Instalación en VS Code
2. Configuración y preferencias
3. Modos de sugerencia: inline, panel, chat
4. Atajos de teclado esenciales
5. Copilot Chat para explicaciones

### Actividades
- Laboratorio: Configuración del entorno
- Ejercicio: Completar funciones con Copilot
- Ejercicio: Usar Copilot Chat para debugging
- Quiz de uso de Copilot

---

## Módulo 03: Ingeniería de Prompts para Código
**Duración: 2 horas**

### Objetivos
- Diseñar prompts efectivos para generación de código
- Aplicar técnicas de few-shot y chain-of-thought
- Iterar y refinar resultados

### Contenidos
1. Anatomía de un buen prompt de código
2. Contexto: importancia de la información
3. Técnicas: zero-shot, few-shot, chain-of-thought
4. Prompts para diferentes lenguajes
5. Iteración y refinamiento

### Actividades
- Ejercicio: De prompt malo a prompt bueno
- Laboratorio: Crear biblioteca de prompts personales
- Práctica: Generar código con diferentes técnicas
- Quiz de técnicas de prompting

---

## Módulo 04: Generación de Código con LLMs
**Duración: 2.5 horas**

### Objetivos
- Generar código funcional con IA
- Evaluar y mejorar código generado
- Integrar código de IA en proyectos reales

### Contenidos
1. Generación de funciones y clases
2. Patrones de diseño con IA
3. Refactoring asistido
4. Conversión entre lenguajes
5. Generación de boilerplate

### Actividades
- Ejercicio: Crear CRUD completo con IA
- Laboratorio: Refactorizar código legacy
- Práctica: Convertir código Python a JavaScript
- Proyecto parcial: Módulo de aplicación

---

## Módulo 05: Debugging Asistido por IA
**Duración: 2 horas**

### Objetivos
- Usar IA para identificar bugs
- Interpretar mensajes de error con IA
- Aplicar técnicas de debugging sistemático

### Contenidos
1. Análisis de errores con IA
2. Prompts para debugging efectivo
3. Rubber duck debugging con IA
4. Análisis de stack traces
5. Debugging de código ajeno

### Actividades
- Laboratorio: Encontrar bugs en código dado
- Ejercicio: Interpretar errores de producción
- Práctica: Debugging de aplicación completa
- Quiz de técnicas de debugging

---

## Módulo 06: Testing con IA
**Duración: 2 horas**

### Objetivos
- Generar tests unitarios con IA
- Crear tests de integración
- Mejorar cobertura de código

### Contenidos
1. Generación de tests unitarios
2. Tests de integración con IA
3. Casos edge y corner cases
4. Mocking y fixtures
5. Análisis de cobertura

### Actividades
- Ejercicio: Tests para funciones existentes
- Laboratorio: Aumentar cobertura de proyecto
- Práctica: Tests end-to-end básicos
- Quiz de conceptos de testing

---

## Módulo 07: Documentación Automatizada
**Duración: 1.5 horas**

### Objetivos
- Generar documentación técnica con IA
- Crear READMEs profesionales
- Documentar APIs automáticamente

### Contenidos
1. Docstrings y comentarios de código
2. README.md profesional
3. Documentación de APIs
4. Diagramas y visualizaciones
5. Changelog y release notes

### Actividades
- Ejercicio: Documentar módulo de código
- Laboratorio: Crear README completo
- Práctica: Documentar API REST
- Quiz de documentación

---

## Módulo 08: Proyecto Integrador
**Duración: 2.5 horas**

### Objetivos
- Aplicar todas las técnicas aprendidas
- Desarrollar mini-proyecto completo
- Presentar y defender trabajo

### Proyecto
Desarrollar una aplicación CLI o web simple que incluya:
- Código generado con IA
- Tests automatizados
- Documentación completa
- Control de versiones

### Entregables
1. Repositorio GitHub con código
2. README documentado
3. Suite de tests
4. Presentación breve (5 min)

### Evaluación
- Funcionalidad: 30%
- Calidad de código: 25%
- Tests: 25%
- Documentación: 20%

## Políticas del Curso

### Asistencia
- Mínimo 80% de asistencia para certificación
- Puntualidad valorada

### Integridad Académica
- El código de IA debe ser comprendido y revisado
- Citar fuentes cuando corresponda
- Colaboración permitida con atribución

### Evaluación
- No hay exámenes formales
- Evaluación continua basada en ejercicios
- Proyecto integrador como evaluación final
