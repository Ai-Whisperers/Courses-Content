# Quiz - Módulo 01
## Asistentes de Código

**Duración:** 5 minutos
**Puntos totales:** 100 (10 puntos cada pregunta)

---

### Pregunta 1

¿Cuál es la diferencia principal entre GitHub Copilot y Claude/ChatGPT para programación?

- A) Copilot es gratuito y Claude es pago
- B) Copilot sugiere código inline mientras escribes, Claude es conversacional
- C) Claude genera mejor código que Copilot
- D) No hay diferencia, son iguales

<details>
<summary>Ver respuesta</summary>

**Respuesta correcta: B) Copilot sugiere código inline mientras escribes, Claude es conversacional**

Copilot se integra en el editor y sugiere código mientras escribes (Tab para aceptar). Claude/ChatGPT son interfaces de chat donde describes lo que necesitas y recibes código como respuesta.
</details>

---

### Pregunta 2

¿Qué técnica mejora las sugerencias de Copilot?

- A) Escribir código lo más rápido posible
- B) Usar nombres de variables cortos como `a`, `b`, `c`
- C) Escribir comentarios descriptivos antes de la función
- D) Evitar usar tipos o JSDoc

<details>
<summary>Ver respuesta</summary>

**Respuesta correcta: C) Escribir comentarios descriptivos antes de la función**

Copilot usa el contexto del archivo para generar sugerencias. Comentarios descriptivos, buenos nombres de funciones, y tipos/JSDoc ayudan al modelo a entender tu intención y generar código más preciso.
</details>

---

### Pregunta 3

En Cursor Editor, ¿qué hace el comando Ctrl+K (Cmd+K)?

- A) Abre el terminal
- B) Guarda el archivo
- C) Abre la edición inline con IA para el código seleccionado
- D) Cierra el archivo actual

<details>
<summary>Ver respuesta</summary>

**Respuesta correcta: C) Abre la edición inline con IA para el código seleccionado**

Ctrl+K permite seleccionar código y describir cómo quieres modificarlo. La IA genera los cambios y muestra un diff que puedes aceptar o rechazar.
</details>

### Pregunta 4

¿Cuándo es MÁS apropiado usar Claude/ChatGPT en lugar de Copilot?

- A) Para completar una línea de código simple
- B) Para obtener sugerencias mientras escribes
- C) Para diseñar arquitectura o debugging complejo
- D) Para generar nombres de variables

<details>
<summary>Ver respuesta</summary>

**Respuesta correcta: C) Para diseñar arquitectura o debugging complejo**

Claude/ChatGPT son mejores para tareas que requieren razonamiento extendido, explicaciones detalladas, o contexto de múltiples archivos. Copilot es mejor para completado rápido inline.
</details>

### Pregunta 5

¿Por qué es importante revisar el código generado por IA antes de usarlo?

- A) Porque la IA siempre genera código incorrecto
- B) Porque puede contener bugs, vulnerabilidades, o código desactualizado
- C) Porque es ilegal usar código de IA sin revisión
- D) Porque el código nunca compila

<details>
<summary>Ver respuesta</summary>

**Respuesta correcta: B) Porque puede contener bugs, vulnerabilidades, o código desactualizado**

La IA puede generar código funcional pero con problemas sutiles: vulnerabilidades de seguridad, versiones antiguas de APIs, edge cases no manejados, o patrones diferentes a tu proyecto.
</details>

### Pregunta 6

¿Qué característica distingue a Cursor Composer de otras herramientas?

- A) Solo funciona con Python
- B) Puede generar/modificar múltiples archivos a la vez
- C) No usa IA
- D) Es más lento que otras herramientas

<details>
<summary>Ver respuesta</summary>

**Respuesta correcta: B) Puede generar/modificar múltiples archivos a la vez**

Cursor Composer (Ctrl+I) permite describir cambios que afectan varios archivos del proyecto. Por ejemplo, crear un modelo, su migración, rutas y tests, todo en una sola acción.
</details>

### Pregunta 7

Un buen prompt para generar código debe incluir:

- A) Solo la tarea a realizar
- B) Contexto, tarea, requisitos específicos, y formato deseado
- C) Una oración muy corta
- D) Solo ejemplos sin explicación

<details>
<summary>Ver respuesta</summary>

**Respuesta correcta: B) Contexto, tarea, requisitos específicos, y formato deseado**

Un prompt completo incluye: stack tecnológico (contexto), qué necesitas (tarea), especificaciones técnicas (requisitos), y cómo quieres recibir el código (formato).
</details>

### Pregunta 8

¿Qué NO deberías hacer con código generado por IA para autenticación o seguridad?

- A) Revisarlo cuidadosamente
- B) Usarlo directamente en producción sin revisión
- C) Agregar tests
- D) Verificar con documentación oficial

<details>
<summary>Ver respuesta</summary>

**Respuesta correcta: B) Usarlo directamente en producción sin revisión**

El código de seguridad (autenticación, criptografía, permisos) generado por IA debe SIEMPRE revisarse manualmente por un experto. Errores sutiles pueden crear vulnerabilidades graves.
</details>

### Pregunta 9

¿Cuál es el flujo de trabajo recomendado para usar IA en desarrollo?

- A) IA solo al final para documentar
- B) Planificación → Scaffolding → Implementación → Debugging → Refactoring
- C) Generar todo el proyecto de una vez
- D) No usar IA nunca

<details>
<summary>Ver respuesta</summary>

**Respuesta correcta: B) Planificación → Scaffolding → Implementación → Debugging → Refactoring**

Un flujo efectivo usa diferentes herramientas para cada fase: Claude para planificar, Cursor Composer para estructura, Copilot para implementación, y vuelta a Claude para debugging/refactoring.
</details>

### Pregunta 10

GitHub Copilot está disponible gratis para:

- A) Todos los usuarios
- B) Solo empresas grandes
- C) Estudiantes con GitHub Education
- D) Solo desarrolladores senior

<details>
<summary>Ver respuesta</summary>

**Respuesta correcta: C) Estudiantes con GitHub Education**

GitHub Copilot es gratuito para estudiantes verificados con GitHub Education. Para otros usuarios cuesta $10/mes individual o $19/mes business.
</details>
