# Quiz - Módulo 02: Backend con IA

**Duración:** 5 minutos | **Puntos:** 100

---

### Pregunta 1
¿Qué debe incluir un buen prompt para diseñar una API?
- A) Solo el nombre del proyecto
- B) Contexto, requisitos funcionales, stack tecnológico, y formato esperado
- C) Solo ejemplos de código
- D) Una descripción muy corta

<details><summary>Respuesta</summary>B - Un prompt completo ayuda a la IA a generar diseños más precisos y útiles.</details>

---

### Pregunta 2
¿Por qué usar Zod o Joi para validación en lugar de validación manual?
- A) Son más lentos
- B) Proporcionan validación tipada, mensajes claros, y se integran con TypeScript
- C) Son obligatorios por ley
- D) La IA no puede generar validación manual

<details><summary>Respuesta</summary>B - Los esquemas de validación son más mantenibles, tipados, y producen mejores mensajes de error.</details>

---

### Pregunta 3
¿Cuál es el código HTTP correcto para "recurso creado exitosamente"?
- A) 200 OK
- B) 201 Created
- C) 204 No Content
- D) 301 Redirect

<details><summary>Respuesta</summary>B - 201 Created es el código estándar para recursos recién creados.</details>

---

### Pregunta 4
¿Para qué sirven los índices en la base de datos?
- A) Hacer la base de datos más bonita
- B) Acelerar consultas en columnas frecuentemente buscadas
- C) Aumentar el espacio en disco
- D) Reemplazar las claves primarias

<details><summary>Respuesta</summary>B - Los índices optimizan búsquedas pero aumentan el espacio y pueden ralentizar escrituras.</details>

---

### Pregunta 5
¿Qué patrón de respuesta es más mantenible?
- A) Retornar datos diferentes en cada endpoint
- B) Estructura consistente: { success, data, error?, meta? }
- C) Solo retornar el objeto sin wrapper
- D) Retornar HTML siempre

<details><summary>Respuesta</summary>B - Una estructura consistente facilita el manejo en el frontend y debugging.</details>
