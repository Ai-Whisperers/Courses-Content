## Competencias a Desarrollar

### Competencia Principal
Integrar herramientas de IA en el ciclo completo de desarrollo de software para aumentar productividad y calidad de código.

### Competencias Específicas

1. **Generación de Código**
   - Usar asistentes de código efectivamente
   - Escribir prompts que produzcan código de calidad
   - Evaluar y mejorar código generado

2. **Arquitectura y Diseño**
   - Diseñar APIs con asistencia de IA
   - Modelar bases de datos
   - Documentar decisiones arquitectónicas

3. **Desarrollo Fullstack**
   - Generar backend y frontend coordinados
   - Integrar componentes de manera coherente
   - Mantener consistencia de código

4. **Calidad y Testing**
   - Generar tests comprehensivos
   - Identificar bugs y vulnerabilidades
   - Realizar code reviews asistidos

5. **DevOps y Deployment**
   - Automatizar procesos de deployment
   - Configurar pipelines CI/CD
   - Troubleshoot problemas de producción

## Metodología de Enseñanza

### Enfoque Práctico
- 70% del tiempo en ejercicios hands-on
- Coding en vivo con participación
- Proyectos incrementales

### Uso de IA Durante el Curso
- Se permite y fomenta uso de IA
- Énfasis en verificar y entender código generado
- Documentar prompts efectivos

### Trabajo Colaborativo
- Pair programming en algunos ejercicios
- Code reviews entre participantes
- Compartir prompts y técnicas

## Sistema de Evaluación

| Componente | Peso | Criterios |
|------------|------|-----------|
| **Ejercicios** | 30% | Completitud, calidad de código |
| **Quizzes** | 20% | Comprensión conceptual |
| **Proyecto Final** | 50% | Funcionalidad, código, documentación |

### Rúbrica del Proyecto Final

| Criterio | Excelente (90-100) | Bueno (70-89) | Regular (50-69) |
|----------|-------------------|---------------|-----------------|
| Funcionalidad | Completa, sin bugs | Mayormente funcional | Funcionalidad parcial |
| Código | Limpio, documentado | Legible | Funciona pero desordenado |
| Testing | >80% cobertura | 50-80% cobertura | <50% cobertura |
| DevOps | Pipeline completo | Pipeline básico | Sin automatización |
| Documentación | README completo | README básico | Sin documentación |

## Políticas del Curso

### Uso de IA
- ✅ Usar IA para generar código
- ✅ Usar IA para debugging
- ✅ Usar IA para documentación
- ⚠️ SIEMPRE revisar código generado
- ⚠️ SIEMPRE entender antes de usar
- ❌ NO copiar sin verificar

### Integridad Académica
- El proyecto final debe ser trabajo original
- Se puede usar IA como herramienta
- Documentar uso de IA en el proceso
