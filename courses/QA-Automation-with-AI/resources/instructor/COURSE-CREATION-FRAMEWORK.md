# AI Whisperers - Course Creation Framework

## Overview

This document provides the framework for creating AI training courses for different professional roles. It defines the methodology, templates, and best practices for delivering consistent, high-quality AI education.

---

## Course Development Methodology

### Phase 1: Audience Analysis

1. **Define the Target Role**
   - Current skills and responsibilities
   - Daily tasks and pain points
   - Tools currently used
   - Learning preferences

2. **Identify Transformation Goal**
   - What new capabilities will they gain?
   - How will their workflow change?
   - What new responsibilities can they take on?
   - What business value will they deliver?

3. **Assess Prerequisites**
   - Technical knowledge required
   - Tools familiarity needed
   - Domain expertise expected
   - Time commitment available

### Phase 2: Learning Objectives

Use the SMART framework:
- **S**pecific: "Use Claude Code to generate unit tests"
- **M**easurable: "Achieve 80% code coverage"
- **A**chievable: "Within 2 weeks of practice"
- **R**elevant: "For their daily testing work"
- **T**ime-bound: "By end of module 6"

### Phase 3: Content Development

For each module:
1. **Theory** (20%) - Concepts and principles
2. **Demonstration** (30%) - Show how it works
3. **Practice** (40%) - Hands-on exercises
4. **Assessment** (10%) - Validate understanding

### Phase 4: Course Materials

Required for each course:
- [ ] Course syllabus
- [ ] Learning objectives per module
- [ ] Presentation slides
- [ ] Code examples
- [ ] Hands-on exercises
- [ ] Assessment rubrics
- [ ] Reference materials
- [ ] Final project specification

## Common Course Components

### 1. AI Setup and Configuration

Every course should include:
- Tool installation (Claude Code, GitHub CLI)
- Authentication setup
- Project configuration
- CLAUDE.md creation

### 2. Context Engineering

Every course should teach:
- The context hierarchy
- Effective prompting
- Information structuring
- Iterative refinement

### 3. Agentic Patterns

Relevant patterns per role:

| Role | Key Patterns |
|------|-------------|
| QA | Reflection, Chaining, Evaluation |
| Developer | Chaining, Tool Use, Planning |
| DevOps | Parallelization, Automation, Monitoring |
| Data Analyst | RAG, Analysis, Visualization |
| Tech Writer | Chaining, Reflection, Editing |
| Security | Guardrails, Evaluation, Safety |

### 4. Quality Assurance

Every course should include:
- Reviewing AI outputs
- Validating accuracy
- Iterative improvement
- Human oversight

### 5. Final Projects

All final projects should require:
- Real-world application
- Complete workflow demonstration
- Documentation
- Presentation/demo

## Assessment Criteria

### Knowledge Assessment (Written)
- Understanding of AI capabilities
- Prompt engineering principles
- Domain-specific concepts

### Skills Assessment (Practical)
- Tool proficiency
- Output quality
- Problem-solving approach
- Iterative improvement

### Project Assessment (Comprehensive)
- Requirements completion
- AI usage effectiveness
- Output quality
- Documentation completeness
- Presentation clarity

### Grading Rubric

| Criterion | Weight | Excellent | Good | Needs Work |
|-----------|--------|-----------|------|------------|
| AI Usage | 25% | Optimal prompts, good context | Adequate prompts | Poor context |
| Output Quality | 30% | Production-ready | Minor issues | Major issues |
| Completeness | 20% | All requirements | Most requirements | Missing items |
| Documentation | 15% | Comprehensive | Adequate | Incomplete |
| Presentation | 10% | Clear, professional | Understandable | Confusing |

## Getting Started

### To Create a New Course:

1. **Choose Target Role** - Who will benefit most?
2. **Define Transformation** - What will they become?
3. **Map Content** - What do they need to learn?
4. **Build Modules** - Follow the standard template
5. **Create Exercises** - Hands-on practice
6. **Design Assessment** - Measure success
7. **Develop Materials** - Slides, examples, docs
8. **Pilot Test** - Run with small group
9. **Refine** - Incorporate feedback
10. **Launch** - Full deployment

## Appendix: Prompt Library for Course Development

### For Module Planning
```
Create a module outline for teaching [skill] to [role].
Include: learning objectives, key concepts, exercises, assessment.
Target duration: [X] hours.
```

### For Exercise Development
```
Create a hands-on exercise for [skill].
Include: setup instructions, step-by-step guide, expected outputs, common issues.
Difficulty: [beginner/intermediate/advanced]
```

### For Assessment Creation
```
Create an assessment for [module/skill].
Include: knowledge questions, practical tasks, evaluation criteria.
Test both understanding and application.
```

### For Content Review
```
Review this course content for:
1. Accuracy
2. Clarity
3. Completeness
4. Engagement
5. Practical applicability

Suggest improvements for each area.
```
