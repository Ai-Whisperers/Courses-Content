# Module 8: Grading Rubric
## Capstone Project Evaluation

---

## Overview

Total Points: **100**
Passing Score: **70%**

---

## Category 1: Technical Excellence (40 points)

### Code Quality (15 points)

| Points | Criteria |
|--------|----------|
| 13-15 | Exceptional: Clean, well-organized, follows best practices, excellent naming, proper abstractions |
| 10-12 | Good: Mostly clean, minor issues, follows conventions, good organization |
| 7-9 | Satisfactory: Functional but has code smells, inconsistent style, some issues |
| 4-6 | Below Average: Multiple issues, hard to follow, poor organization |
| 0-3 | Unsatisfactory: Messy, hard to maintain, significant problems |

**Specific Checkpoints:**
- [ ] Consistent code style (2 pts)
- [ ] Meaningful variable/function names (2 pts)
- [ ] Proper error handling (3 pts)
- [ ] No code smells (3 pts)
- [ ] Appropriate abstractions (3 pts)
- [ ] No linting errors (2 pts)

### Testing (15 points)

| Points | Criteria |
|--------|----------|
| 13-15 | Exceptional: 90%+ coverage, edge cases covered, well-organized tests |
| 10-12 | Good: 70-89% coverage, most cases covered, clear test structure |
| 7-9 | Satisfactory: 50-69% coverage, basic cases covered |
| 4-6 | Below Average: <50% coverage, minimal tests |
| 0-3 | Unsatisfactory: Few or no tests, tests don't pass |

**Specific Checkpoints:**
- [ ] Unit tests present (4 pts)
- [ ] Tests pass consistently (3 pts)
- [ ] Edge cases tested (3 pts)
- [ ] Code coverage meets target (3 pts)
- [ ] Test organization (2 pts)

### Security (10 points)

| Points | Criteria |
|--------|----------|
| 9-10 | Exceptional: All security best practices, no vulnerabilities, proactive security |
| 7-8 | Good: Security scan clean, input validation, proper authentication |
| 5-6 | Satisfactory: Basic security measures, minor issues |
| 3-4 | Below Average: Some security concerns, missing validations |
| 0-2 | Unsatisfactory: Significant vulnerabilities, secrets exposed |

**Specific Checkpoints:**
- [ ] No hardcoded secrets (3 pts)
- [ ] Input validation (2 pts)
- [ ] Security scan passes (3 pts)
- [ ] Proper authentication (if applicable) (2 pts)

## Penalty Deductions

| Violation | Deduction |
|-----------|-----------|

## Evaluator Notes

### What We're Looking For

**Technical:**
- Clean, maintainable code
- Proper testing practices
- Security awareness
- Working software

**AI Usage:**
- Strategic tool selection
- Effective prompting
- Understanding of generated code
- Documented process

**Professional:**
- Clear documentation
- Effective communication
- Self-awareness and reflection
- Time management

### Red Flags

- Code clearly not understood by presenter
- AI-generated code accepted without review
- Security vulnerabilities
- Missing tests for core functionality
- Incomplete or missing documentation
- Unable to explain design decisions
