## Questions

### Q1: AI is best for which architecture task?
- A) Final decision making
- B) Technology comparisons and trade-offs
- C) Budget allocation
- D) Team management

### Q2: What should architecture prompts include?
- A) Just the feature name
- B) Requirements, constraints, and context
- C) Only technical details
- D) Just "design this"

### Q3: What's an ADR?
- A) Automated Design Review
- B) Architecture Decision Record
- C) Application Development Report
- D) Agile Design Reference

### Q4: When evaluating technologies, AI cannot account for:
- A) Feature comparison
- B) Team's existing skills
- C) Performance benchmarks
- D) Community support

### Q5: What's technical debt?
- A) Money owed to developers
- B) Shortcuts that need future work
- C) Database size
- D) Lines of code

### Q6: Best approach for AI architecture suggestions?
- A) Accept without review
- B) Reject all AI input
- C) Validate with team and constraints
- D) Only use for small projects

### Q7: What pattern helps with multiple payment types?
- A) Singleton
- B) Strategy
- C) Observer
- D) Factory

### Q8: For migrating monolith to microservices, AI suggests:
- A) Rewrite everything at once
- B) Phased incremental approach
- C) Abandon the project
- D) Keep the monolith forever

### Q9: AI architecture advice should be documented in:
- A) Emails only
- B) ADRs and design docs
- C) Nowhere
- D) Slack messages

### Q10: What's a key limitation of AI for architecture?
- A) Can't generate diagrams
- B) Doesn't know your specific business context
- C) Can't compare technologies
- D) Only works with JavaScript

### Q11: Good refactoring prompt includes:
- A) Current state and target state
- B) Just "refactor this"
- C) Only the bugs
- D) Team member names

### Q12: When should you use AI for system design?
- A) Never
- B) As initial brainstorming and validation
- C) Only for final decisions
- D) Only for small systems

### Q13: What's needed to validate AI architecture?
- A) Nothing
- B) Senior engineer review
- C) AI validation
- D) Customer approval

### Q14: AI can help identify which code issue?
- A) Political problems
- B) Design pattern opportunities
- C) Budget overruns
- D) Team conflicts

### Q15: Best use of AI in technical debt discussion?
- A) Let AI decide priorities
- B) Use AI to identify and categorize debt
- C) Ignore AI completely
- D) Only use for new projects

## Answers

| Q | A | Reason |
|---|---|--------|
| 1 | B | AI excels at comparisons |
| 2 | B | Context enables better suggestions |
| 3 | B | Architecture Decision Record |
| 4 | B | AI doesn't know your team |
| 5 | B | Shortcuts needing future work |
| 6 | C | Validate with constraints |
| 7 | B | Strategy for varying algorithms |
| 8 | B | Phased approach is safer |
| 9 | B | Proper documentation |
| 10 | B | Lacks specific business context |
| 11 | A | Current and target states |
| 12 | B | Brainstorming and validation |
| 13 | B | Senior engineer review |
| 14 | B | Pattern opportunities |
| 15 | B | Identify and categorize |
