# Errores Comunes con IA - Investigación y Academia

## Guía para Evitar los Errores Más Frecuentes

Este documento recopila los errores más comunes que cometen los estudiantes e investigadores al usar IA para trabajos académicos, con ejemplos específicos y cómo evitarlos.

---

## ⚠️ ADVERTENCIA ÉTICA

**El mal uso de IA en academia puede tener consecuencias graves:**

- Plagio: Anulación del trabajo
- Invención de citas: Pérdida de credibilidad permanente
- Datos falsos: Retractación de publicaciones
- Dependencia excesiva: No desarrollas habilidades críticas

**La IA es una herramienta de apoyo, no un sustituto del pensamiento crítico.**

## Error #1: Citas Bibliográficas Inventadas

### El Problema
La IA genera referencias que parecen reales pero NO EXISTEN.

### Ejemplo Real
```
IA genera:
"Según González y Martínez (2021) en su estudio 'Impacto de
las redes sociales en adolescentes paraguayos', publicado en
la Revista Paraguaya de Psicología, vol. 15(3), pp. 45-67..."

Verificación:
- No existe "Revista Paraguaya de Psicología"
- Autores con esos apellidos existen, pero no ese paper
- El DOI generado no funciona
- Esta cita es 100% INVENTADA
```

### Por Qué Es Grave
- Es falsificación académica
- Puede causar reprobación o expulsión
- Destruye tu credibilidad profesional
- Afecta a tu director de tesis

### Cómo Evitarlo
1. **NUNCA usar una cita sin verificarla**
2. **Buscar en Google Scholar** el título exacto
3. **Verificar el DOI** en doi.org
4. **Acceder al paper** y leer al menos el abstract
5. **Marcar como "POR VERIFICAR"** hasta confirmar

## Error #2: Parafraseo que Sigue Siendo Plagio

### El Problema
Parafrasear superficialmente manteniendo la estructura original.

### Ejemplo Real
```
Texto original:
"El impacto de las redes sociales en la autoestima de los
adolescentes ha sido ampliamente estudiado, mostrando una
correlación negativa significativa."

Parafraseo de IA (TODAVÍA ES PLAGIO):
"El efecto de las plataformas sociales en la autoestima de
los jóvenes ha sido extensamente investigado, revelando una
asociación negativa importante."

Problema: Misma estructura, mismas ideas, solo sinónimos
```

### Parafraseo Correcto
```
Correcto (con comprensión real):
"Diversos estudios (García, 2020; López, 2021) han encontrado
que el tiempo de uso de redes sociales se relaciona inversamente
con indicadores de autoestima en población adolescente,
particularmente cuando el uso supera las 3 horas diarias."

Diferencia:
- Agrega fuentes propias
- Incluye detalles específicos
- Demuestra comprensión
- No es solo reordenamiento de palabras
```

### Cómo Evitarlo
1. **Leer el texto original varias veces**
2. **Cerrar el documento** y escribir con tus palabras
3. **Agregar tu análisis** o interpretación
4. **Siempre citar** la fuente original
5. **Usar herramientas anti-plagio** antes de entregar

## Error #3: Estadísticas y Datos Falsos

### El Problema
La IA inventa números que suenan plausibles.

### Ejemplo Real
```
IA genera:
"Según el censo del INE (2022), el 67.3% de los hogares
paraguayos tiene acceso a internet de banda ancha."

Verificación en INE/DGEEC:
- El dato real (EPH 2022) es 52.4%
- No se llama "internet de banda ancha" en el reporte
- Hay diferencia urbano/rural significativa no mencionada
```

### Datos que la IA Frecuentemente Inventa
- Porcentajes y estadísticas demográficas
- Cifras económicas (PIB, inflación, etc.)
- Resultados de estudios
- Fechas de eventos históricos

### Fuentes Confiables para Paraguay
| Tema | Fuente oficial |
|------|----------------|
| Población/demografía | INE (antes DGEEC) |
| Economía | BCP (Banco Central) |
| Educación | MEC |
| Salud | MSPBS |
| Comercio | CAPECO |

### Cómo Evitarlo
1. **Verificar CADA número** en fuentes primarias
2. **Documentar la fuente** con link y fecha de acceso
3. **Capturar pantalla** del dato en la fuente
4. **Marcar datos como "VERIFICAR"** hasta confirmar

## Error #4: Estructura de Marco Teórico Inadecuada

### El Problema
Aceptar estructuras genéricas sin adaptarlas al tema específico.

### Ejemplo Real
```
IA genera para cualquier tema:
2. MARCO TEÓRICO
   2.1. Antecedentes
   2.2. Bases teóricas
   2.3. Marco conceptual

Problema: Es tan genérico que no ayuda a organizar el trabajo
```

### Estructura Mejor
```
Para tesis sobre comercio electrónico en PyMEs:

2. MARCO TEÓRICO

2.1. El comercio electrónico
    2.1.1. Concepto y evolución
    2.1.2. Modalidades (B2B, B2C, C2C)
    2.1.3. Estado en Paraguay (datos CAPECO)

2.2. Las PyMEs como objeto de estudio
    2.2.1. Definición según ley paraguaya
    2.2.2. Importancia económica
    2.2.3. Retos de digitalización

2.3. Teorías de adopción tecnológica
    2.3.1. Modelo TAM (Davis, 1989)
    2.3.2. Marco TOE (Tornatzky, 1990)
    2.3.3. Modelo a utilizar en esta investigación

2.4. Factores determinantes de adopción
    2.4.1. Factores tecnológicos
    2.4.2. Factores organizacionales
    2.4.3. Factores del entorno

2.5. Antecedentes de investigación
    2.5.1. Estudios internacionales
    2.5.2. Estudios regionales
    2.5.3. Estudios en Paraguay
```

### Cómo Evitarlo
1. **Partir de tu pregunta de investigación**
2. **Identificar conceptos clave** que necesitan definición
3. **Organizar de general a específico**
4. **Terminar conectando con tu estudio**

## Error #5: Metodología Copiada sin Entender

### El Problema
Usar descripciones metodológicas genéricas sin adaptar.

### Ejemplo Real
```
IA genera:
"Se utilizará un enfoque cuantitativo de tipo descriptivo-
correlacional con diseño no experimental transversal."

Problema: ¿El estudiante sabe qué significa cada término?
- ¿Por qué cuantitativo y no cualitativo?
- ¿Por qué correlacional?
- ¿Qué implica "no experimental"?
```

### Preguntas a Responder
| Decisión | Justificación necesaria |
|----------|------------------------|
| Enfoque | ¿Por qué cuanti/cuali/mixto? |
| Alcance | ¿Por qué descriptivo/correlacional/explicativo? |
| Diseño | ¿Por qué experimental/no experimental? |
| Temporalidad | ¿Por qué transversal/longitudinal? |
| Muestreo | ¿Cómo calculaste el tamaño? |

### Cómo Evitarlo
1. **Estudiar cada término** antes de usarlo
2. **Justificar cada decisión** metodológica
3. **Conectar con objetivos** de investigación
4. **Consultar con tutor** antes de avanzar

## Error #6: Análisis de Datos Sin Conocimiento

### El Problema
Pedir análisis estadístico sin entender los supuestos.

### Ejemplo Real
```
Estudiante: "Haceme un análisis de correlación de estos datos"

IA: "La correlación de Pearson entre X e Y es r = 0.75"

Problemas ignorados:
- ¿Los datos son normales? (supuesto de Pearson)
- ¿La relación es lineal?
- ¿Hay outliers afectando?
- ¿El tamaño de muestra es suficiente?
- ¿r = 0.75 es estadísticamente significativo?
```

### Checklist para Análisis Estadístico
- [ ] ¿Verificaste normalidad? (Shapiro-Wilk, histograma)
- [ ] ¿Revisaste outliers?
- [ ] ¿El test es apropiado para tus datos?
- [ ] ¿Reportaste significancia (p-value)?
- [ ] ¿Calculaste tamaño del efecto?
- [ ] ¿Puedes interpretar el resultado?

### Cómo Evitarlo
1. **Aprender estadística básica** antes de analizar
2. **Verificar supuestos** de cada test
3. **Usar software apropiado** (SPSS, JASP, R)
4. **Consultar con estadístico** si es necesario

## Error #7: Conclusiones que No Siguen de los Resultados

### El Problema
Escribir conclusiones que van más allá de lo que los datos muestran.

### Ejemplo Real
```
Resultado: "Se encontró correlación positiva (r=0.45) entre
uso de redes sociales y ansiedad en 120 universitarios de Asunción"

Conclusión incorrecta (de la IA):
"Las redes sociales CAUSAN ansiedad en los jóvenes paraguayos"

Problemas:
- Correlación ≠ causalidad
- Muestra de Asunción ≠ todos los paraguayos
- Universitarios ≠ todos los jóvenes
- r=0.45 es moderado, no fuerte
```

### Conclusión Correcta
```
"Los resultados sugieren una asociación moderada entre el
tiempo de uso de redes sociales y los niveles de ansiedad
en estudiantes universitarios de Asunción. Esta relación,
aunque estadísticamente significativa, no implica causalidad
y debe interpretarse considerando las limitaciones del diseño
transversal. Se recomienda investigación longitudinal para
explorar posibles relaciones causales."
```

### Cómo Evitarlo
1. **Nunca afirmar causalidad** sin diseño experimental
2. **Limitar generalización** a la muestra estudiada
3. **Reconocer limitaciones** explícitamente
4. **Usar lenguaje tentativo**: "sugiere", "indica", "parece"

## Error #8: Formato APA Inconsistente

### El Problema
Mezclar formatos o aplicar APA incorrectamente.

### Errores Comunes
```
❌ Incorrecto (APA 7):
"Según García, M. (2021)..."  → No va inicial del nombre en texto
"(García, 2021, pág. 45)"    → Es "p." no "pág."
"Op. cit."                   → No existe en APA
"Ibídem"                     → No existe en APA

✅ Correcto:
"Según García (2021)..."
"(García, 2021, p. 45)"
Repetir la cita completa cada vez
```

### Formato APA 7 Rápido
| Tipo | Formato |
|------|---------|
| 1 autor | (García, 2021) |
| 2 autores | (García & López, 2021) |
| 3+ autores | (García et al., 2021) |
| Cita directa | (García, 2021, p. 45) |
| Cita en texto | García (2021) afirma... |
| Comunicación personal | (M. García, comunicación personal, 1 de marzo de 2024) |

### Cómo Evitarlo
1. **Usar Zotero o Mendeley** para gestionar referencias
2. **Configurar estilo APA 7** en el gestor
3. **Revisar manual APA** ante dudas
4. **Verificar consistencia** en todo el documento

## Error #9: Resúmenes de Papers Sin Leer el Paper

### El Problema
Confiar en resúmenes de IA sin leer la fuente original.

### Ejemplo Real
```
Estudiante: "Resumime este paper [título]"

IA genera un resumen que:
- Puede confundir papers con títulos similares
- Inventa resultados que suenan lógicos
- Omite limitaciones importantes
- Malinterpreta la metodología

Estudiante lo cita en su tesis sin leer el original
```

### Lo Que Puede Salir Mal
- El paper dice lo opuesto a lo que la IA resumió
- Citas fuera de contexto
- Metodología incompatible con tu estudio
- El paper no existe (inventado)

### Cómo Evitarlo
1. **Siempre acceder al paper original**
2. **Leer al menos**: abstract, metodología, conclusiones
3. **Tomar notas propias** mientras lees
4. **Usar IA solo para orientación** inicial

## Error #10: No Distinguir Fuentes Primarias y Secundarias

### El Problema
Citar autores a través de otros sin acceder al original.

### Ejemplo Real
```
La IA genera:
"Según la teoría de Bandura (1977), citado en García (2020)..."

Problema:
- ¿García citó bien a Bandura?
- ¿El contexto original era otro?
- Deberías leer a Bandura directamente
```

### Cuándo es Aceptable Citar Secundaria
- El texto original es inaccesible (muy antiguo, idioma raro)
- Es una cita dentro de un trabajo mayor
- Se indica claramente: "Bandura, 1977, como se citó en García, 2020"

### Cómo Evitarlo
1. **Buscar la fuente primaria** siempre que sea posible
2. **Si no es posible, indicar** "como se citó en"
3. **Para autores clásicos** (Piaget, Bandura, etc.), leer originales
4. **Documentar el esfuerzo** de búsqueda

## Error #11: Hipótesis que No Son Hipótesis

### El Problema
Formular hipótesis que no son testeables o están mal redactadas.

### Ejemplo Real
```
❌ Incorrectas:
"Las redes sociales son malas" → Juicio de valor, no hipótesis
"Tal vez haya relación" → Muy vago
"Los jóvenes usan mucho el celular" → Descripción, no hipótesis

✅ Correctas:
"Existe una correlación positiva entre el tiempo diario de uso
de redes sociales y los niveles de ansiedad en estudiantes
universitarios de la FPUNA."

"Los estudiantes que usan redes sociales más de 3 horas diarias
presentan mayores niveles de ansiedad que aquellos que las usan
menos de 1 hora."
```

### Características de Buena Hipótesis
- [ ] Es testeable empíricamente
- [ ] Establece relación entre variables
- [ ] Es específica y medible
- [ ] Se deriva del marco teórico
- [ ] Puede ser falsificada

### Cómo Evitarlo
1. **Partir de la teoría**, no de intuiciones
2. **Definir operacionalmente** cada variable
3. **Especificar dirección** de la relación
4. **Consultar con tutor** antes de avanzar

## Error #12: No Documentar el Uso de IA

### El Problema
Usar IA sin transparencia, lo cual puede considerarse deshonestidad académica.

### Políticas Actuales
Muchas universidades ahora requieren:
- Declarar el uso de herramientas de IA
- Especificar cómo se utilizaron
- Indicar qué partes fueron asistidas por IA

### Cómo Documentar (Ejemplo)
```
DECLARACIÓN DE USO DE IA

En la elaboración de este trabajo se utilizaron las siguientes
herramientas de inteligencia artificial:

1. Claude (Anthropic) - para:
   - Búsqueda inicial de literatura
   - Revisión de gramática
   - Sugerencias de estructura

2. Verificación realizada:
   - Todas las citas fueron verificadas en fuentes originales
   - Los datos estadísticos fueron corroborados en fuentes oficiales
   - La redacción final es de autoría propia

La responsabilidad del contenido recae en el autor.
```

### Cómo Evitarlo
1. **Verificar política** de tu universidad/facultad
2. **Documentar uso** de forma transparente
3. **Guardar historial** de conversaciones con IA
4. **Asumir responsabilidad** por el contenido final

## Checklist Anti-Errores para Academia

### Para Citas y Referencias
- [ ] ¿Verifiqué cada cita en la fuente original?
- [ ] ¿Los DOIs funcionan?
- [ ] ¿El formato es APA 7 consistente?
- [ ] ¿Accedí al paper o solo al abstract?

### Para Datos
- [ ] ¿Los números vienen de fuentes oficiales?
- [ ] ¿Documenté la fuente con fecha de acceso?
- [ ] ¿Verifiqué con fuentes secundarias?
- [ ] ¿Capturé pantalla de respaldo?

### Para Análisis
- [ ] ¿Entiendo el test estadístico usado?
- [ ] ¿Verifiqué los supuestos?
- [ ] ¿Puedo interpretar los resultados?
- [ ] ¿Las conclusiones siguen de los datos?

### Para Ética
- [ ] ¿Documenté el uso de IA?
- [ ] ¿Cumplo con las políticas de mi institución?
- [ ] ¿El trabajo refleja MI comprensión?
- [ ] ¿Estoy siendo transparente?
