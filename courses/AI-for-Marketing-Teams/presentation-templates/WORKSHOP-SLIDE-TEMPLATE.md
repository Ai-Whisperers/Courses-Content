## Workshop Structure

Workshops follow this flow:
1. Brief intro (5 min)
2. Demo (10 min)
3. Guided practice (15-20 min)
4. Independent practice (20-30 min)
5. Share and discuss (10-15 min)

## Workshop Facilitation Tips

### Before Workshop
- [ ] Test all tools with actual exercise
- [ ] Prepare backup examples
- [ ] Have troubleshooting guide ready
- [ ] Set up screen sharing

### During Workshop
- Walk the room during working time
- Identify common struggles early
- Keep time visible for participants
- Have helpers assist stuck participants

### Common Issues
| Problem | Solution |
|---------|----------|
| Tool not working | Pair with neighbor |
| Stuck on step | Review demo |
| Output too long | Show how to constrain |
| Output too generic | Add more context |

### Timing Buffer
Build in 5-10 minutes buffer per workshop for:
- Technical issues
- Extended questions
- Slower participants
- Rich discussions

---

*Workshops are where learning happens. Prioritize hands-on time over lecture.*
