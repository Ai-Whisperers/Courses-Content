## Document Principles

This documentation follows:

- **Single source of truth** - One place for each topic
- **Clear hierarchy** - Guides > Templates > Reference
- **Minimal redundancy** - No duplicate content
- **Focused files** - Each file covers one topic
- **Easy navigation** - Clear naming, READMEs everywhere
