# Course Quality Checklist

> Comprehensive review checklist before course launch

---

## Module Completeness

### For Each Module

#### Structure (10 items)
- [ ] Module header with duration, level, prerequisites
- [ ] 4-6 learning objectives using action verbs
- [ ] 2-3 paragraph overview explaining value
- [ ] 4-6 well-organized content sections
- [ ] Best practices section (5-8 items)
- [ ] Common pitfalls section (5-6 items)
- [ ] Knowledge check questions (3-5)
- [ ] Summary with key takeaways
- [ ] Preview of next module
- [ ] Additional resources listed

#### Content Quality (10 items)
- [ ] 300-450 lines of content
- [ ] Sufficient depth for session duration
- [ ] All technical information accurate
- [ ] Examples use current tools/versions
- [ ] Real-world, relatable scenarios
- [ ] 5+ industries represented in examples
- [ ] Actionable frameworks included
- [ ] Every concept has business takeaway
- [ ] Visual elements (tables, lists) present
- [ ] Technical terms explained when first used

#### Writing Style (8 items)
- [ ] Professional but approachable tone
- [ ] Uses second person ("you")
- [ ] Confidence-building language
- [ ] Rhetorical questions for engagement
- [ ] Mix of short and long sentences
- [ ] No paragraphs longer than 6-7 lines
- [ ] Active voice (80%+)
- [ ] Consistent terminology

---

## Exercise Checklist

### For Each Exercise
- [ ] Clear objective stated
- [ ] Prerequisites listed
- [ ] 4-5 progressive tasks
- [ ] Time estimate per task
- [ ] Success criteria defined
- [ ] Hints for common issues
- [ ] Submission checklist
- [ ] Challenge task (optional)

---

## Quiz Checklist

### For Each Quiz
- [ ] 10-15 questions
- [ ] Mix of question types
- [ ] Clear question stems
- [ ] Plausible answer options
- [ ] Answer key complete
- [ ] Explanations for answers
- [ ] Difficulty balanced (60/30/10)
- [ ] Covers all key concepts

---

## Technical Verification

### Code Examples
- [ ] All code runs without errors
- [ ] Expected outputs documented
- [ ] Dependencies listed
- [ ] Comments explain key parts
- [ ] No hardcoded sensitive data
- [ ] Follows language conventions

### Links & References
- [ ] All internal links work
- [ ] External links valid
- [ ] Resources accessible
- [ ] No broken images

---

## Course-Level Checklist

### Documentation
- [ ] README.md complete and accurate
- [ ] SYLLABUS.md reflects actual content
- [ ] Learning outcomes match content
- [ ] Prerequisites clearly stated
- [ ] Duration estimates realistic

### Assessment
- [ ] Assessment weights defined
- [ ] Grading rubrics complete
- [ ] Answer keys available
- [ ] Capstone requirements clear
- [ ] Submission process documented

### Resources
- [ ] Cheatsheet covers key topics
- [ ] Glossary includes all terms
- [ ] Prompts library populated
- [ ] Templates ready for use
- [ ] AI tool configs complete

---

## Pre-Launch Final Check

### Content Review
- [ ] All modules complete
- [ ] Consistent quality throughout
- [ ] No placeholder text remaining
- [ ] Cross-references accurate

### Technical Test
- [ ] All code examples tested
- [ ] All links verified
- [ ] All downloads work
- [ ] No broken formatting

### User Experience
- [ ] Clear navigation
- [ ] Logical progression
- [ ] Appropriate difficulty curve
- [ ] Engaging content
