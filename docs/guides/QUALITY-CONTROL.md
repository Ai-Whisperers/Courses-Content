# Quality Control Checklist for AI-Generated Course Content

**Purpose:** Ensure consistent, high-quality content across all AI-assisted course development
**Usage:** Review every module, exercise, and assessment before final approval
**Target:** Match or exceed Module 1 quality standard

---

## How to Use This Checklist

1. **Complete immediately after Claude generates content** (while context is fresh)
2. **Mark items with:**
   - ✅ Pass - Meets standard
   - ⚠️ Review - Needs minor adjustment
   - ❌ Fail - Requires significant revision
3. **Aim for 90%+ Pass rate** before proceeding
4. **Document issues** for pattern recognition and prompt improvement

### Quality Thresholds

| Score | Rating | Action |
|-------|--------|--------|
| **95-100%** | Excellent | Approve - Ready for production |
| **90-94%** | Very Good | Minor polish - Proceed with caution |
| **80-89%** | Good | Review/iteration needed before approval |
| **70-79%** | Fair | Significant revision required |
| **<70%** | Poor | Regenerate with improved prompts |

**Target:** All modules should score 90%+ before final approval

## Action Items Based on Review

### If Score is 80-89%:
**Required Actions:**
1. Document specific failing items
2. Create targeted improvement prompts for Claude
3. Regenerate or manually edit problem sections
4. Re-review with this checklist
5. Confirm score improves to 90%+

### If Score is <80%:
**Required Actions:**
1. Analyze prompt quality - what was unclear?
2. Improve context and reference materials
3. Regenerate entire section with better prompt
4. Consider if SME input needed for technical accuracy
5. Document lessons learned for future prompts

## Best Practices for Using This Checklist

### 1. Review Immediately
- Complete checklist right after Claude generates content
- Context is fresh, issues easier to spot
- Faster iteration cycle

### 2. Be Objective
- Don't lower standards because "it's AI-generated"
- Module 1 is the benchmark - match or exceed it
- Business professionals deserve high quality

### 3. Document Patterns
- Track common issues across multiple modules
- Identify prompt improvements needed
- Build institutional knowledge

### 4. Iterate Systematically
- Fix highest-impact issues first
- One category at a time
- Re-check after each revision

### 5. Maintain Standards
- Don't approve content below 90%
- Quality compounds - excellent modules build excellent courses
- Better to spend extra hour now than deal with issues later

## Continuous Improvement

### After Each Module, Update:

**Prompt Library:**
- What prompts produced high scores?
- What wording avoided common issues?
- What context improved accuracy?

**Quality Patterns:**
- What consistently scores well?
- What consistently needs revision?
- What varies by module type?

**Efficiency Metrics:**
- Time per module (prompt + review + revision)
- Iterations needed
- Pass rate on first attempt
