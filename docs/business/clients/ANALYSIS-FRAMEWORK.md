# Target Client Analysis Framework

> **Purpose**: A systematic approach to identifying, analyzing, and designing courses for specific client segments. Use this framework before creating any new course.

---

## Part 1: Client Segmentation Matrix

### Primary Segmentation Dimensions

| Dimension | Categories | Indicators |
|-----------|------------|------------|
| **Industry** | Tech, Finance, Healthcare, Manufacturing, Retail, Education, Government | Company type, regulatory requirements, tech maturity |
| **Role Level** | Individual Contributor, Team Lead, Manager, Director, Executive | Title, decision-making scope, budget authority |
| **Technical Depth** | Non-technical, Semi-technical, Technical, Expert | Current tools used, programming knowledge, system understanding |
| **Organization Size** | Startup (<50), SMB (50-500), Enterprise (500+) | Employee count, process formality, tool budget |
| **Learning Context** | Upskilling, Reskilling, New Role, Compliance, Competitive Advantage | Motivation, urgency, sponsor type |
