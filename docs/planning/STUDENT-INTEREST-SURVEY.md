## FOR ADMINISTRATORS: ANALYSIS GUIDE

### Priority Metrics to Track:

1. **Track Preference Distribution**
   - Calculate % QA vs Web Dev preference
   - Identify "no preference" students
   - Determine minimum viable class sizes

2. **Skill Level Distribution**
   - Beginner vs Intermediate vs Advanced
   - Programming language distribution
   - Prior AI tool experience

3. **Time Commitment**
   - Average expected hours/week
   - Identify students who might struggle with time

4. **Career Alignment**
   - Does track preference match career interest?
   - Project excitement alignment with tracks

5. **Concerns & Expectations**
   - Common worries to address in orientation
   - Skills students want to master
   - Logistical needs

### Decision Points:

**If QA preference < 30%**: Consider offering only Web Dev track this year
**If Web Dev preference < 30%**: Consider offering only QA track this year
**If 30-70% split**: Proceed with both tracks as planned
**If strong 3rd track interest emerges**: Consider DevOps or Data track for future
