# Quiz - Módulo 04: Testing y QA con IA

**Duración:** 5 minutos | **Puntos:** 100

---

### Pregunta 1
¿Qué son los "edge cases" en testing?
- A) Casos que nunca ocurren
- B) Casos límite como null, vacío, valores extremos
- C) Tests que fallan siempre
- D) Tests de rendimiento

<details><summary>Respuesta</summary>B - Edge cases son situaciones límite que pueden causar comportamiento inesperado.</details>

---

### Pregunta 2
¿Cuál es una buena cobertura de tests para código de producción?
- A) 10%
- B) 50%
- C) 80% o más
- D) 100% es obligatorio

<details><summary>Respuesta</summary>C - 80% es un buen objetivo; 100% no siempre es práctico o necesario.</details>

---

### Pregunta 3
¿Qué es un "mock" en testing?
- A) Un test que falla
- B) Una simulación de una dependencia externa
- C) Un test manual
- D) Un error del código

<details><summary>Respuesta</summary>B - Los mocks simulan comportamiento de dependencias para aislar el código bajo test.</details>

---

### Pregunta 4
¿Qué tipos de vulnerabilidades evalúa OWASP Top 10?
- A) Solo virus
- B) Vulnerabilidades web comunes como inyección, XSS, auth rota
- C) Problemas de hardware
- D) Errores de tipeo

<details><summary>Respuesta</summary>B - OWASP Top 10 cubre las vulnerabilidades de seguridad web más críticas.</details>

---

### Pregunta 5
¿Por qué es importante hacer code review con IA?
- A) Reemplaza completamente la revisión humana
- B) Identifica problemas rápidamente que podrían pasarse por alto
- C) Es obligatorio por ley
- D) Solo sirve para formatear código

<details><summary>Respuesta</summary>B - La IA es un asistente que complementa (no reemplaza) el review humano.</details>
