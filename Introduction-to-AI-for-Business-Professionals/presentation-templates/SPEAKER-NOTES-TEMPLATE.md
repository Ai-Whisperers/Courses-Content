## General Speaking Guidelines

### Delivery Techniques

**Voice & Pace**
- Speak clearly and at moderate pace
- Pause after key points
- Vary tone to maintain interest
- Project enthusiasm
- Use natural speaking voice

**Body Language (In-Person)**
- Make eye contact with audience
- Use hand gestures naturally
- Move around (don't stand rigidly)
- Face audience, not screen
- Smile and show approachability

**Virtual Presentation**
- Look at camera, not screen
- Use virtual hand raise features
- Monitor chat actively
- Use screen annotations
- Share reactions and emojis

### Engagement Strategies

**Ask Questions**
- "Who here has experience with...?"
- "What do you think will happen if...?"
- "Can anyone share an example of...?"
- Wait 5-7 seconds for responses
- Acknowledge all contributions

**Check Understanding**
- "Does this make sense so far?"
- "Any questions before we move on?"
- "Let's do a quick understanding check..."
- Use polls or hand raises
- Watch for confused expressions

**Encourage Participation**
- Call on volunteers
- Use breakout rooms
- Invite chat comments
- Share personal mistakes
- Create safe space for questions

## Module-Specific Speaking Notes

### Module 1: Introduction to AI-Assisted Development

**PRE-SESSION PREP**
- Have Claude Code open and ready for live demo
- Prepare example codebase for exploration
- Test AI response times
- Have backup screenshots

**KEY MESSAGES TO EMPHASIZE**
1. AI augments, doesn't replace QA thinking
2. Always validate AI output
3. Context is crucial for quality results
4. Iteration improves AI responses

**DEMONSTRATIONS**
Demo 1: First AI interaction
- Start with simple question
- Show how AI responds
- Demonstrate follow-up questions
- Highlight conversation flow

Demo 2: Code explanation
- Open complex function
- Ask AI to explain
- Show how to ask for clarification
- Compare AI explanation with manual analysis

**COMMON STUDENT CONCERNS**
1. "Will AI replace my job?"
   - No, it makes you more valuable
   - Focus shifts to strategy and validation
   - Human judgment remains essential

2. "What if AI makes mistakes?"
   - It will! That's expected
   - Your role: catch and correct
   - Shows importance of QA skills

3. "How do I know what to trust?"
   - Always validate against documentation
   - Run code to verify it works
   - Use multiple sources

**TIMING BREAKDOWN**
- Introduction: 5 min
- Concept explanation: 15 min
- Live demos: 20 min
- Student exploration: 15 min
- Q&A: 5 min
Total: 60 minutes

**ENERGY MANAGEMENT**
- Start high energy to engage
- Demos are natural energy peaks
- Check energy at 30-min mark
- End with enthusiasm for next module

---

### Module 2: Context Engineering

**PRE-SESSION PREP**
- Prepare example CLAUDE.md files
- Have good/bad prompt examples ready
- Test prompt variations
- Show real impact of context

**KEY MESSAGES TO EMPHASIZE**
1. Quality of output = Quality of input + Context
2. Context hierarchy matters
3. Iterative refinement is normal
4. Document successful patterns

**DEMONSTRATIONS**
Demo 1: Same prompt, different contexts
- Show generic prompt result
- Add context, show improvement
- Add constraints, show refinement
- Compare all three outputs

Demo 2: Building a CLAUDE.md file
- Start from blank file
- Add project info progressively
- Show AI output improvement
- Highlight what to include/exclude

**INTERACTIVE EXERCISES**
Exercise 1: Prompt improvement (10 min)
- Give students weak prompt
- Ask them to improve it
- Share results
- Discuss what worked

Exercise 2: Write CLAUDE.md (15 min)
- Students create for sample project
- Pair-share for feedback
- Instructor reviews best examples

**COMMON MISTAKES TO ADDRESS**
1. Too vague: "Write tests"
   Better: "Write Jest unit tests for the UserService class focusing on authentication methods"

2. Missing context: Not mentioning frameworks
   Include: "Using React 18, TypeScript, and Testing Library"

3. No output format specified
   Add: "Output as separate test files with descriptive names"

**TIMING BREAKDOWN**
- Recap Module 1: 3 min
- Context concepts: 12 min
- Demo 1: 10 min
- Demo 2: 10 min
- Interactive exercises: 25 min
Total: 60 minutes

## Handling Difficult Situations

### Technical Difficulties

**Demo Fails**
- Stay calm and acknowledge
- Use backup screenshots
- Explain what should happen
- Offer to show later
- Move on quickly

**Student Can't Follow Along**
- Don't hold up entire class
- Offer to help during break
- Direct to teaching assistant
- Provide catch-up resources
- Pair with advanced student

**Tool Version Mismatch**
- Acknowledge the difference
- Show workaround if possible
- Update documentation note
- Continue with your version
- Offer office hours help

### Student Questions

**Off-Topic Question**
- Acknowledge interest
- Note for later discussion
- Redirect to current topic
- Offer to discuss after class
- "Great question, let's table that..."

**Repeated Questions**
- Show patience, not frustration
- Rephrase explanation differently
- Use different example
- Ask class if someone can explain
- Check if broader confusion

**Don't Know Answer**
- Be honest: "Great question, I don't know"
- Offer to research and follow up
- Ask if anyone in class knows
- Look it up together if time permits
- Document for course improvement

### Engagement Issues

**Low Energy/Participation**
- Take a stretch break
- Switch to interactive activity
- Ask direct questions
- Share energetic story
- Adjust pace if too slow

**One Person Dominating**
- Thank for contributions
- "Let's hear from others..."
- Use directed questions
- Private conversation if persists
- Balance attention fairly

**Confused Expressions**
- Stop and check understanding
- Ask specific clarifying questions
- Revisit prerequisites
- Provide additional examples
- Slow down pace

## Time Management

### Running Behind

**Priority Framework**
1. MUST COVER (Critical concepts)
   - Core learning objectives
   - Prerequisites for next module
   - Essential hands-on practice

2. SHOULD COVER (Important but flexible)
   - Extended examples
   - Additional best practices
   - Bonus tips

3. NICE TO COVER (Supplementary)
   - Advanced topics
   - Extra demonstrations
   - Extended Q&A

**Recovery Strategies**
- Shorten Q&A sections
- Reduce number of examples
- Speed up slides without interaction
- Skip optional demos
- Assign some content as reading

### Running Ahead

**Extension Options**
- Deeper Q&A
- Additional examples
- Preview next module
- Advanced techniques
- Student showcase
- Extra practice time

## Presentation Day Script

### Opening (5 minutes)

"Good [morning/afternoon], everyone! Welcome to [Course/Module Name].

My name is [Name], and I'll be your instructor today.

Before we begin, let me share a bit about my background...
[Brief 30-second bio highlighting relevant experience]

Today's session is all about [main topic]. By the end, you'll be able to [key outcomes].

Here's how we'll structure our time:
- [Time block 1]: [Activity]
- [Time block 2]: [Activity]
- [Break time]
- [Time block 3]: [Activity]

Feel free to ask questions anytime - just [raise hand/unmute/use chat].

Let's get started!"

### Transitions Between Topics

**Standard Transition**
"Excellent work on [previous topic]. Now let's build on that by exploring [next topic]..."

**Callback Transition**
"Remember when we discussed [earlier concept]? Now we're going to see how that connects to [new topic]..."

**Question Transition**
"So far we've covered [summary]. What do you think happens when [introduce new scenario]?"

### Handling Breaks

**Before Break**
"Great progress so far! Let's take a [X]-minute break.

Before you go:
- Save your work
- [Any specific instructions]
- We'll resume at [specific time]

See you in [X] minutes!"

**After Break**
"Welcome back! Let's do a quick recap of where we were...
[2-sentence summary]

Any questions from the first half before we continue?"

### Closing (5 minutes)

"Fantastic work today, everyone!

Let's recap what we covered:
- [Key point 1]
- [Key point 2]
- [Key point 3]

You should now be able to [restate learning objectives].

Next steps:
1. [Action item 1]
2. [Action item 2]
3. [Action item 3]

Resources:
- [Link/document 1]
- [Link/document 2]

Questions? [Take final questions]

Thank you for your attention and participation. See you in [next module]!"

## Self-Improvement Questions

After each presentation, reflect on:

**Content**
- Were learning objectives achieved?
- Did examples resonate with students?
- What concepts caused confusion?
- What additional examples would help?

**Delivery**
- Was pace appropriate?
- Did I engage students effectively?
- Were transitions smooth?
- Did I manage time well?

**Interaction**
- Did students ask questions?
- How well did I handle questions?
- Were students engaged throughout?
- Did everyone participate?

**Technical**
- Did demos work smoothly?
- Were visuals clear?
- Was audio/video quality good?
- Any tool issues?

## Emergency Backup Plans

### Total Tech Failure
**Plan B: No-Slides Presentation**
- Have outline in notes
- Draw diagrams on whiteboard/paper
- Focus on discussion
- Provide slides afterward
- Turn challenge into engagement

### Can't Access Code/Demo
**Plan C: Walkthrough Without Execution**
- Describe what code does
- Show screenshots
- Ask students to visualize
- Assign as homework
- Schedule follow-up demo

### Few Students Attend
**Plan D: Intimate Session**
- More personalized attention
- Hands-on help
- Tailored examples
- Extended Q&A
- Individual feedback

### Wrong Room/Time Issues
**Plan E: Quick Adaptation**
- Announce correct location
- Start slightly late
- Summarize for latecomers
- Record from start
- Send detailed notes

## Instructor Tips from Experience

### Before You Start Teaching
1. **Know your audience** - Survey students beforehand
2. **Practice out loud** - Not just reading slides
3. **Test everything twice** - Demos, links, downloads
4. **Have backups** - Screenshots, offline resources
5. **Prepare more than you'll use** - Better to cut than stretch

### During Teaching
1. **Watch the clock** - Visible timer helps
2. **Read the room** - Adjust based on energy/confusion
3. **Pause frequently** - Give time to absorb
4. **Repeat key points** - Repetition aids learning
5. **Stay positive** - Enthusiasm is contagious

### After Teaching
1. **Don't dwell on mistakes** - Note and move on
2. **Celebrate wins** - Student success is your success
3. **Iterate constantly** - Every session improves
4. **Ask for feedback** - Students provide valuable insights
5. **Rest and recharge** - Teaching is energy-intensive
