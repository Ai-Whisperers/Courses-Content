# /review

Analiza el código seleccionado o los cambios recientes buscando:

1. Errores lógicos o de sintaxis.
2. Adherencia a los estándares definidos en `CLAUDE.md`.
3. Oportunidades de refactorización para mejorar legibilidad.
4. Reducción de deuda técnica.

# /fix-lint

Busca errores de linting y propón correcciones automáticas manteniendo el estilo del proyecto.

# /generate-test

Crea un archivo de prueba en `tests/` para el componente o función seleccionada siguiendo el patrón AAA.
