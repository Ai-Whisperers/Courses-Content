## Common Challenges & Solutions

### Challenge 1: Scope Creep
**Problem:** Trying to do too much
**Solution:** Focus on minimum requirements first, then enhance

### Challenge 2: AI Not Helping
**Problem:** Poor quality AI outputs
**Solution:** Review CLAUDE.md, improve prompts, iterate more

### Challenge 3: Coverage Stuck Below 75%
**Problem:** Can't reach coverage target
**Solution:** Use AI to identify untested paths, prioritize critical code

### Challenge 4: Time Management
**Problem:** Running out of time
**Solution:** Focus on P0 tests first, use 3-week timeline

### Challenge 5: Test Quality Issues
**Problem:** Tests passing but not effective
**Solution:** Apply quality checklist, get peer review

### Challenge 6: CI/CD Problems
**Problem:** Pipeline failing
**Solution:** Test locally first, check workflow syntax, review logs

## Pro Tips

### From Successful Students:

1. **Start with the 3-Week Plan**
   - Don't try to finish in one week
   - Space out work for better quality
   - Leave buffer time for issues

2. **Invest in CLAUDE.md Early**
   - Spend 45-60 minutes on this
   - Every prompt afterward benefits
   - Update it as you learn the project

3. **Prioritize P0 Tests First**
   - Get critical paths tested
   - Ensures minimum viable coverage
   - Reduces stress if time runs short

4. **Use AI for Grunt Work**
   - Let AI generate boilerplate
   - You focus on test logic and assertions
   - Review and refine everything

5. **Document as You Go**
   - Don't leave documentation for last
   - Capture prompts that work well
   - Note lessons learned in real-time

6. **Get Feedback Early**
   - Share work-in-progress
   - Fix issues incrementally
   - Don't wait until the end

7. **Make Your Demo Compelling**
   - Show before/after comparisons
   - Highlight interesting AI interactions
   - Share genuine insights
